# Система учёта оборудования — Автотесты (pytest + Selenium)

## Описание

Django-приложение для учёта оборудования с CRUD-операциями, поиском, фильтрацией по статусу и человекопонятными URL (slug).

Модели:
- **Supplier** (Поставщик) — название, email, телефон
- **Equipment** (Оборудование) — название, slug, инвентарный номер, описание, статус, поставщик

## Установка и запуск

### 1. Установить зависимости

```bash
pip install -r requirements.txt
```

### 2. Применить миграции

```bash
python manage.py makemigrations inventory
python manage.py migrate
```

### 3. Запустить приложение (опционально)

```bash
python manage.py runserver
```

Приложение будет доступно по адресу http://127.0.0.1:8000/

### 4. Запустить все тесты

```bash
pytest
```

Или по отдельности:

```bash
pytest tests/test_models.py -v     # Модели
pytest tests/test_forms.py -v      # Формы
pytest tests/test_views.py -v      # Представления
pytest tests/test_selenium.py -v   # Selenium (нужен Chrome)
```

## Требования для Selenium-тестов

- Google Chrome (установлен в системе)
- ChromeDriver скачивается автоматически через `webdriver-manager`
- Тесты запускаются в headless-режиме (без видимого окна браузера)

## Покрытие тестами

### Модульные тесты (pytest) — 56 тестов

**Модели:**
- Создание объектов с корректными данными
- Метод `__str__`
- Уникальность полей (inventory_number, name, slug)
- Обязательность полей
- Максимальная длина текстовых полей
- Автогенерация slug из названия (транслитерация)
- Каскадное поведение ForeignKey (PROTECT)
- Автоматические поля created_at, updated_at

**Формы:**
- Валидность при полном наборе данных
- Невалидность при пропуске обязательных полей
- Проверка уникальности (дубликат инвентарного номера / названия поставщика)
- Опциональные поля

**Представления:**
- Доступность URL (status 200)
- Корректный шаблон
- Наличие объектов в контексте
- Создание/обновление/удаление через POST
- 404 для несуществующих объектов
- Поиск по названию и инвентарному номеру
- Фильтрация по статусу
- Пустой поиск возвращает все записи

### Функциональные тесты (Selenium) — 13 тестов

**Навигация:**
- Переход на главную, проверка навигационного меню
- Переход по ссылкам меню

**CRUD оборудования:**
- Создание через форму
- Просмотр детальной страницы
- Редактирование
- Удаление + проверка 404
- Кнопка «Отмена»

**Связанные сущности:**
- Новый поставщик появляется в выпадающем списке формы
- Удаление поставщика с оборудованием заблокировано (PROTECT)

**Поиск и фильтрация через интерфейс:**
- Поиск по названию
- Фильтр по статусу

**Slug (дополнительная функция):**
- Автогенерация slug при создании
- Работа CRUD через slug-URL
